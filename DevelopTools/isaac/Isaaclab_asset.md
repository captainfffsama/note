# Articulation

`class isaaclab.assets.Articulation` 基类 `AssetBase`

这是一个用于表示关节的资产类别。

一个“关节结构”（articulation）是由多个通过关节相互连接的刚性部件组成的系统。这些关节可以是固定的，也可以是可运动的（即能够被驱动来改变位置的）。关节的类型多种多样，例如旋转关节（revolute joint）、直线运动关节（prismatic joint）等。目前，该关节结构模型主要使用旋转关节和直线运动关节进行测试；该模型同时支持“固定底座型”（fixed-base）和“浮动底座型”（floating-base）两种结构类型。关节结构的类型是由其“根关节”（root joint）的性质决定的：如果根关节是固定的，则整个关节结构属于“固定底座型”；否则属于“浮动底座型”。这一信息可以通过 ` ` Articulation.is_fixed_base ` ` 属性来查询

要使某个资产被视为一个“关节结构”（即一个能够实现特定运动的组件），该资产的根节点必须满足特定的条件（例如：其坐标值必须以 “USDArticulationRootAPI.” 的格式表示）。该 API 用于使用“简化坐标公式”来定义该关节结构的子结构。在运行模拟时，物理引擎会解析这些根节点的数据，并在物理引擎中创建相应的关节结构。该关节结构的根节点可以通过 “ `AssetBaseCfg.prim_path` ” 属性进行指定。

该“关节系统建模”类还提供了通过自定义执行器模型来增强关节系统仿真功能的功能。这些执行器模型可以是显式的（explicit），也可以是隐式的（implicit），具体细节请参见 `isaaclab.actuators` 模块中的说明。执行器模型的相关信息是通过 `ArticulationCfg.actuators` 属性来指定的；在仿真运行时，这些模型会被解析并用于初始化相应的执行器组件。

在仿真过程中，关节控制类首先会应用相应的执行器模型，根据用户指定的目标来计算出相应的关节控制指令（即需要关节执行的动作）。这些控制指令随后会被直接输入到仿真系统中进行执行。这些控制指令可以是关于关节位置的指令、关于关节速度的指令，也可以是关于关节所需施加力的指令。以下代码片段展示了如何使用这些控制指令来实现对关节位置的控制：

```python
# an example instance of the articulation class
my_articulation = Articulation(cfg)

# set joint position targets
my_articulation.set_joint_position_target(position)
# propagate the actuator models and apply the computed commands into the simulation
my_articulation.write_data_to_sim()

# step the simulation using the simulation context
sim_context.step()

# update the articulation state, where dt is the simulation time step
my_articulation.update(dt)
```
